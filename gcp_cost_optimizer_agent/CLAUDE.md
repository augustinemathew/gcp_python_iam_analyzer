# CLAUDE.md

GCP Cost Optimizer Agent — a Vertex AI Agent Engine agent that answers natural language
questions about GCP resource inventory and billing costs.

## Why this exists

Augustine needed an agent that can:
- Survey all resources in a GCP project (Cloud Asset Inventory)
- Query billing data to find expensive services and SKUs (BigQuery billing export)
- List running VMs with machine types for rightsizing analysis
- Answer queries like "What's expensive?", "How much am I spending?", "What resources do I have?"

It also serves as a real-world test case for iamspy's Agent Engine pkl extraction feature
(see ~/gcp_ae). The agent is deployed to the same project used for iamspy development.

## Tools

### `list_resources(project_id, asset_types=None)`
Calls Cloud Asset Inventory. Returns all resources grouped by type with counts.
Answers: "What do I have?", "How many VMs?", "List all buckets."

### `list_running_vms(project_id)`
Calls Compute Engine aggregated list. Returns running VMs grouped by zone with machine type and IPs.
Answers: "What VMs are running?", "List my instances."

### `get_billing_summary(project_id, billing_table, days=30)`
Queries the BigQuery billing export table. Returns total cost, cost by service, and top 20 SKUs.
The user must provide the billing export table name (e.g. `my-billing.dataset.gcp_billing_export_v1_XXXX`).
Answers: "How much am I spending?", "What are my most expensive services?", "Show me billing for the last 7 days."

## Adding a new tool

1. Create `agent/tools/<name>.py` with a single public function
2. Good docstring — AG2 uses it as the tool description for the LLM
3. Import and add to the `tools=[...]` list in `agent/build.py`
4. Add the pip package to `requirements.txt` and `REQUIREMENTS` in `deploy.py`
5. Re-deploy with `python deploy.py`

## IAM permissions needed

The Agent Engine runtime identity needs:
- `roles/cloudasset.viewer` — for list_resources
- `roles/compute.viewer` — for list_running_vms
- `roles/bigquery.jobUser` — for get_billing_summary (run queries)
- `roles/bigquery.dataViewer` — for get_billing_summary (read billing export tables)

Grant on the project before querying deployed agent. `deploy.py` grants these automatically.

## iamspy integration

After deploying, the pkl is at `gs://augtestbucket/cost_optimizer_agent/agent_engine.pkl`.
To scan the deployed agent for IAM permissions:

```bash
# In ~/gcp_ae:
gcloud storage cp gs://augtestbucket/cost_optimizer_agent/agent_engine.pkl /tmp/cost_optimizer.pkl
# Then extract + scan (iamspy pkl extraction — feature in progress)
iamspy scan-pkl /tmp/cost_optimizer.pkl
```
